# SIEM Introduction  
*TryHackMe – SOC Analyst Level 1*

## Overview

This lab introduced Security Information and Event Management (SIEM) systems. SIEM platforms are central to SOC operations because they aggregate logs from multiple sources and generate alerts based on suspicious activity.

I explored how SIEM tools help analysts detect and investigate threats efficiently.

---

## Objectives

- Understand what a SIEM is
- Learn how alerts are generated
- Perform basic alert investigation

---

## What is a SIEM?

A SIEM system:
- Collects logs from endpoints, servers, and network devices
- Correlates events
- Generates alerts
- Provides dashboards for monitoring

It improves visibility across the organisation’s infrastructure.

---

## Alert Investigation Process

When a SIEM generates an alert, analysts must:

1. Review event details  
2. Identify affected systems  
3. Determine severity  
4. Check related logs  
5. Decide whether to escalate  

---

## Key Concepts Learned

### False Positives
Not all alerts indicate real threats. Analysts must carefully validate alerts before escalation.

### Log Correlation
SIEM platforms correlate multiple events to detect patterns that may indicate malicious behaviour.
